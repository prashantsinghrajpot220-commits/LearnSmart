## App Icon

### Specifications

| Property | Value |
|----------|-------|
| **Dimensions** | 512x512 pixels |
| **Format** | PNG with no transparency (solid background) |
| **Safe Zone** | All important graphics within 72x72dp center area |
| **Style** | Flat, no shadows, no text |
| **Primary Color** | Sage Green (#9CAF88) |
| **Background** | Charcoal (#2C3E50) |
| **Corner Radius** | 20% (for modern rounded look) |

### Design Guidelines

#### Visual Elements
- **Logo/Icon:** Educational symbol (book, graduation cap, or abstract learning icon)
- **Color Scheme:**
  - Primary: Sage Green (#9CAF88)
  - Secondary: Warm Sand (#F5F1E8)
  - Accent: Charcoal (#2C3E50)
- **Style:** Minimalist, modern, professional
- **Contrast:** High contrast for visibility on all backgrounds

#### Do's ✅
- Use vector-based design for scalability
- Ensure icon is recognizable at small sizes (48x48dp)
- Use solid colors without gradients
- Center the main graphic
- Maintain consistent brand colors

#### Don'ts ❌
- Don't include text in the icon
- Don't use shadows or 3D effects
- Don't use transparency
- Don't use complex details
- Don't include photos

### File Locations

```
/assets/icon.png
```

### Density-Specific Versions

For Android, the icon should be available in multiple densities:

| Density | Resolution | Location |
|---------|------------|----------|
| MDPI | 48x48dp | /android/app/src/main/res/mipmap-mdpi/ic_launcher.png |
| HDPI | 72x72dp | /android/app/src/main/res/mipmap-hdpi/ic_launcher.png |
| XHDPI | 96x96dp | /android/app/src/main/res/mipmap-xhdpi/ic_launcher.png |
| XXHDPI | 144x144dp | /android/app/src/main/res/mipmap-xxhdpi/ic_launcher.png |
| XXXHDPI | 192x192dp | /android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png |

**Note:** The base 512x512 icon is automatically resized by Expo build tools.

## Adaptive Icon

### Specifications

| Property | Value |
|----------|-------|
| **Foreground Dimensions** | 512x512 pixels |
| **Background Color** | Warm Sand (#F5F1E8) |
| **Foreground Style** | Same as app icon |
| **Mask Layer** | Generated by Expo |

### Design Guidelines

#### Foreground Image
- Contains the main app icon/logo
- Transparent background (except for the icon itself)
- Centered within the safe zone
- Must look good when clipped to different shapes

#### Background
- Solid color: Warm Sand (#F5F1E8)
- No patterns or gradients
- Provides contrast with foreground

### Android Adaptive Icon Layers

Adaptive icons consist of two layers:

1. **Foreground Layer:** Main icon/logo
2. **Background Layer:** Solid color

The Android system combines these layers and applies a mask (circle, squircle, rounded square, etc.) based on the device manufacturer's launcher.

### File Locations

```
/assets/adaptive-icon.png (foreground image)
```

### Configuration in app.json

```json
{
  "android": {
    "adaptiveIcon": {
      "foregroundImage": "./assets/adaptive-icon.png",
      "backgroundColor": "#F5F1E8"
    }
  }
}
```

## Splash Screen

### Specifications

| Property | Value |
|----------|-------|
| **Dimensions** | 1080x1920 pixels (baseline for mobile) |
| **Format** | PNG |
| **Background** | Warm Sand (#F5F1E8) |
| **Logo Size** | 256x256 pixels |
| **App Name Text** | "LearnSmart" |
| **Style** | Minimal, professional, no animations |
| **Resize Mode** | Contain |

### Design Guidelines

#### Layout

```
┌─────────────────────────┐
│                         │
│                         │
│         (Logo)          │
│      256x256 px         │
│                         │
│      LearnSmart         │
│    (App Name Text)      │
│                         │
│                         │
└─────────────────────────┘
```

#### Visual Elements
1. **Logo:** Center-aligned, 256x256 pixels
   - Same as app icon/logo
   - Sage Green (#9CAF88) primary color
   - Minimalist design

2. **App Name Text:** Below logo
   - Font: Sans-serif (system default)
   - Size: 32sp
   - Color: Charcoal (#2C3E50)
   - Weight: Bold

3. **Background:** Solid Warm Sand (#F5F1E8)
   - No gradients
   - No patterns
   - No transparency

#### Behavior
- Displays on app startup
- Shows until app is fully loaded
- No animated elements
- Smooth transition to first screen
- No white/blank screens

### File Locations

```
/assets/splash.png
```

### Configuration in app.json

```json
{
  "splash": {
    "image": "./assets/splash.png",
    "resizeMode": "contain",
    "backgroundColor": "#F5F1E8"
  }
}
```

### Multiple Screen Sizes

The splash screen should be optimized for various screen sizes:

| Device Type | Resolution | Status |
|-------------|------------|--------|
| Mobile (Portrait) | 1080x1920 | ✅ Baseline |
| Tablet (Portrait) | 2048x2732 | ✅ Scaled |
| Large Tablet | 2732x2048 | ✅ Scaled |

**Note:** Expo's splash screen API handles scaling automatically.

## Asset Locations

### Directory Structure

```
/assets/
├── icon.png              # Main app icon (512x512)
├── adaptive-icon.png     # Android adaptive icon foreground (512x512)
├── splash.png            # Splash screen (1080x1920)
├── favicon.png           # Web favicon (32x32)
├── animations/          # Animation assets (Lottie, etc.)
└── avatars/            # User avatar assets
```

### Configuration Files

All assets are referenced in `app.json`:

```json
{
  "expo": {
    "name": "LearnSmart",
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#F5F1E8"
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#F5F1E8"
      }
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
```

## Play Store Requirements

### Google Play Console Asset Specifications

#### App Icon (Play Store Listing)
- **Size:** 512x512 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB
- **Color Space:** sRGB
- **Safe Zone:** 80% of the center area

#### Feature Graphic (Store Listing)
- **Size:** 1024x500 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB
- **Requirement:** Optional but recommended

#### Phone Screenshots
- **Size:** At least 1080x1920 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB each
- **Quantity:** Minimum 2 screenshots

#### 7" Tablet Screenshots
- **Size:** At least 1200x1920 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB each
- **Quantity:** Optional

#### 10" Tablet Screenshots
- **Size:** At least 1800x2880 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB each
- **Quantity:** Optional

#### Promo Image
- **Size:** 180x120 pixels
- **Format:** PNG or JPG
- **File Size:** Max 1024 KB
- **Requirement:** Optional

### Asset Best Practices

1. **High Quality:** Use high-resolution images for all assets
2. **Consistent Branding:** Maintain consistent colors and style across all assets
3. **Test on Devices:** Preview assets on actual devices before submission
4. **Follow Guidelines:** Adhere to Google Play Store asset guidelines
5. **Optimize File Size:** Compress images without losing quality

## Asset Management

### Updating Assets

1. **Replace Files:** Replace existing asset files in `/assets/` directory
2. **Clear Cache:** Clear Metro bundler cache if needed:
   ```bash
   npx expo start -c
   ```
3. **Rebuild:** Rebuild the app for production:
   ```bash
   eas build --platform android
   ```

### Testing Assets

1. **Splash Screen:** Verify splash screen displays correctly on app launch
2. **App Icon:** Check app icon appears correctly in device launcher
3. **Adaptive Icon:** Test adaptive icon on different Android devices (Samsung, Google, etc.)
4. **Favicon:** Verify favicon displays in web browser tabs

## Troubleshooting

### Splash Screen Not Showing

**Problem:** Splash screen doesn't display or shows white screen

**Solutions:**
1. Verify splash.png exists in `/assets/` directory
2. Check `app.json` configuration is correct
3. Ensure image format is PNG
4. Verify background color matches splash screen background
5. Clear cache and rebuild:
   ```bash
   npx expo start -c
   ```

### App Icon Not Updating

**Problem:** App icon doesn't update after replacement

**Solutions:**
1. Uninstall the app from device/emulator
2. Clear app data
3. Rebuild the app
4. Restart device/emulator

### Adaptive Icon Issues

**Problem:** Adaptive icon looks distorted or cut off

**Solutions:**
1. Ensure foreground image has transparent background
2. Keep important elements within safe zone (center 66%)
3. Test on multiple devices with different mask shapes
4. Verify background color is solid (no transparency)
